# 提示词模板库

> 存放常用的 Prompt 模板，提升 AI 协作效率

---

## 📁 目录结构

```
prompts/
├── README.md           ← 本文件（模板库说明）
├── development/        ← 开发相关提示词
├── review/             ← 代码审查提示词
├── documentation/      ← 文档编写提示词
└── debugging/          ← 调试排错提示词
```

---

## 🚀 快速使用

### 复制模板 → 填充变量 → 发送给 AI

模板中的 `[变量]` 需要替换为实际内容。

---

## 📝 通用模板

### 1. 功能开发模板

```
## 任务
实现 [功能名称]

## 上下文
当前阶段：@.agent/.current-stage
项目规则：@.agent/rules/00-project-overview.md
阶段规则：@.agent/rules/stages/development.md
领域规则：@.agent/rules/[01-frontend.md 或 02-backend.md]

## 需求描述
[详细描述功能需求]

## 技术要求
- [技术要求1]
- [技术要求2]

## 期望输出
- [ ] 代码实现
- [ ] 单元测试
- [ ] 使用说明
```

### 2. 代码审查模板

```
## 任务
审查以下代码

## 代码
[粘贴代码或 @文件路径]

## 审查维度
参考规则：@.agent/rules/01-frontend.md

请检查：
1. 代码质量和可读性
2. 类型安全
3. 错误处理
4. 性能问题
5. 安全漏洞

## 输出格式
- 问题列表（按严重程度排序）
- 改进建议
- 修改后的代码
```

### 3. Bug 修复模板

```
## 问题描述
[描述 bug 现象]

## 复现步骤
1. [步骤1]
2. [步骤2]

## 期望行为
[期望的正确行为]

## 实际行为
[实际的错误行为]

## 相关代码
@[相关文件路径]

## 请求
1. 分析 bug 原因
2. 提供修复方案
3. 生成修复代码
4. 添加防止复发的测试
```

### 4. 重构模板

```
## 任务
重构 [模块/函数名称]

## 当前代码
@[文件路径]

## 重构目标
- [目标1：提升可读性]
- [目标2：提升性能]
- [目标3：提升可测试性]

## 约束条件
- 保持 API 兼容
- 不改变外部行为
- 遵循规则：@.agent/rules/01-frontend.md

## 期望输出
1. 重构后的代码
2. 变更说明
3. 测试用例
```

### 5. 文档编写模板

```
## 任务
为 [模块/项目] 编写文档

## 文档类型
[README / API 文档 / 使用指南 / 架构文档]

## 参考规则
@.agent/rules/04-documentation.md

## 目标读者
[开发者 / 用户 / 运维人员]

## 需要包含的内容
- [ ] 概述
- [ ] 快速开始
- [ ] 详细说明
- [ ] 示例代码
- [ ] 常见问题

## 代码/模块位置
@[相关文件或目录]
```

## 🔧 场景化模板

### 阶段切换模板

```
项目即将从 [当前阶段] 切换到 [目标阶段]。

当前阶段规则：@.agent/rules/stages/[当前阶段].md
目标阶段规则：@.agent/rules/stages/[目标阶段].md
切换清单：@.agent/rules/99-落地实施规范.md

请帮我：
1. 检查当前阶段目标是否达成
2. 列出需要调整的规则和配置
3. 生成阶段切换检查清单
4. 更新 CHANGELOG.md 条目
```

### 规则评审模板

```
请帮我评审本月的规则执行效果。

## 当前规则
@.agent/rules/

## 问题记录
@.agent/feedback/rule-issues.md

## 相关数据
- Pre-commit 通过率：[X]%
- 主要失败原因：[描述]

## 请分析
1. 哪些规则需要放宽？
2. 哪些规则需要加强？
3. 是否需要新增规则？
4. 如何提升规则执行率？
```

---

## 💡 使用技巧

1. **组合使用**：模板可以组合，例如 功能开发 + 代码审查
2. **自定义**：根据项目特点修改模板
3. **版本控制**：模板也应该纳入版本管理
4. **团队共享**：好的模板应该分享给团队
