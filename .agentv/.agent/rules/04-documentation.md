# 文档编写规则

> 适用于本项目的 Markdown 文档编写

## 1. 文档结构规则

### 1.1 必须包含

- ✅ 标题（H1）
- ✅ 目录（如果超过3个章节）
- ✅ 更新日期

### 1.2 层级规范

- H1：文档标题（只有1个）
- H2：主要章节
- H3：子章节
- H4：细分内容

## 2. 格式规则

### 2.1 代码块

- ✅ 指定语言
- ✅ 添加注释说明
- ✅ 区分正确/错误示例

```typescript
// ✅ 正确示例
const good = "example";

// ❌ 错误示例
const bad = "example";
```

### 2.2 表格

- ✅ 对齐列
- ✅ 表头清晰
- ✅ 内容简洁

### 2.3 列表

- ✅ 使用 emoji 区分类型
- ✅ 保持一致的格式
- ✅ 控制嵌套层级（最多3级）

## 3. 命名规则

### 3.1 文件命名

- 序号前缀：`01-`, `02-`
- 中文名称
- 括号标注：`（技巧1）`

### 3.2 示例

```
01-GLM-4.7 开发者指南.md
03-规则模板配合使用指南（技巧1）.md
```

## 4. 内容规则

- ✅ 提供可执行的代码示例
- ✅ 使用图表说明复杂概念
- ✅ 保持简洁，避免冗余
