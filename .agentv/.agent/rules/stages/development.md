# 开发阶段规则

**阶段标识**：DEVELOPMENT
**目标**：完善功能，保证质量
**预计时长**：4-8 周

---

## 规则优先级

### 🟢 高优先级（必须遵循）

- ✅ 代码必须通过 TypeScript 编译
- ✅ 禁止使用 `any` 类型
- ✅ 新功能必须包含单元测试
- ✅ API 接口必须有参数验证
- ✅ 提交信息符合 Conventional Commits
- ✅ 错误处理必须完整

### 🟡 中优先级（建议遵循）

- ⚠️ 组件应有使用示例
- ⚠️ 复杂逻辑应有注释
- ⚠️ 性能敏感代码需要优化
- ⚠️ API 应有文档

### 🔴 低优先级（时间充裕时）

- ⏸️ 完善的 JSDoc 文档
- ⏸️ 国际化支持
- ⏸️ E2E 测试

---

## 测试要求

- 整体覆盖率 ≥ 70%
- 核心逻辑覆盖率 ≥ 90%

---
