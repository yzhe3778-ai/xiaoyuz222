# 计划创建阶段

根据我们的完整交流，生成一个 markdown 计划文档。

## 计划要求

- 包括清晰、最小、简洁的步骤
- 使用这些表情符号跟踪每个步骤的状态：
  - 🟩 完成
  - 🟨 进行中
  - 🟥 待办
- 在顶部包含整体进度百分比的动态跟踪
- 不要在明确澄清的细节之外添加额外的范围或不必要的复杂性
- 步骤应该是模块化的、优雅的、最小的，并与现有代码库无缝集成

## Markdown 模板

```markdown
# 功能实现计划

**整体进度：** `0%`

## 简要描述
我们要构建的内容及其原因的简短总结。

## 关键决策
在探索过程中做出的关键架构/实现选择：
- 决策 1：[选择] - [简要理由]
- 决策 2：[选择] - [简要理由]

## 任务：
- [ ] 🟥 **步骤 1：[名称]**
  - [ ] 🟥 子任务 1
  - [ ] 🟥 子任务 2
- [ ] 🟥 **步骤 2：[名称]**
  - [ ] 🟥 子任务 1
  - [ ] 🟥 子任务 2
```

## 再次提醒

现在还不是构建的时候。只需编写清晰的计划文档。不要在我们讨论的内容之外添加额外的复杂性或额外的范围。
